## 📱 О проекте

PWA (Progressive Web App) для автоматизации выдачи скидочных карт в розничных точках. Система позволяет клиентам получать виртуальные карты лояльности через смартфон без установки приложения из AppStore/Google Play.

### Преимущества

- **Для клиентов:**
  - Установка в пару кликов через браузер
  - Не занимает место в памяти телефона
  - Всегда с собой в смартфоне
  - Мгновенная регистрация через SMS

- **Для бизнеса:**
  - Исключение фейковых номеров
  - Единая база клиентов
  - Экспорт данных для маркетинга
  - Работа без интернета (офлайн-режим)

---

## 🔑 Ключевые функции

* **PWA (Progressive Web App):** Установка на смартфон в один клик без сторов.
* **SMS-верификация:** Защита от регистрации фейковых номеров.
* **Авторизация по звонку (Check Call):** ✅ NEW Автоматическая верификация через входящий звонок без ввода кода.
* **QR-генерация:** Динамическое создание кода клиента на стороне браузера (qrcode.js).
* **Сессии:** Автоматический вход при повторном открытии (30 дней).
* **Админ-панель:** Просмотр клиентов, экспорт в CSV, поиск по телефону.
* **Инструкция по установке:** Всплывающая подсказка для iOS и Android.

## 📊 Сценарии использования (User Flow)

```
Открытие приложения
    ↓
Есть ли активная сессия (cookie)?
    ├── ДА → Проверка в БД
    │         ├── Пользователь есть → Сразу показать карту
    │         └── Пользователя нет → Показать форму регистрации
    │
    └── НЕТ → Показать форму регистрации (имя + телефон)
              ↓
              Выбрать метод авторизации:
              ├── SMS → Отправка SMS с кодом → Ввод кода → Верификация → Cookie (30 дней) → Карта
              └── Звонок (Check Call) → Автоматический звонок → Ответ на звонок → Верификация → Cookie (30 дней) → Карта
```

**Сценарий 1: Новый пользователь (SMS)**

3. Ввёл код → Карта с QR-кодом

1. Открыл приложение → Автоматический вход

**Сценарий 4: Выход и повторный вход**
1. Нажал "Выйти" → Сессия удалена
2. Открыл приложение → Форма регистрации
3. Ввёл телефон → SMS с кодом ИЛИ Звонок
4. Прошёл верификацию → Карта

**Сценарий 5: Установка PWA**
1. Открыл инструкцию (синий блок)
2. iOS: «Поделиться» → «Добавить на экран домой»
3. Android: «⋮» → «Добавить на гл. экран»

## 🔐 Безопасность

- **HttpOnly Cookie** — Токен сессии недоступен для JavaScript (защита от XSS)
- **Secure Cookie** — Передача только по HTTPS
- **SameSite=Lax** — Защита от CSRF-атак
- **Срок сессии** — 30 дней, затем требуется повторная авторизация
- **SMS-код** — 4 цифры, истекает через 5 минут
- **Токен сессии** — 256 бит энтропии (secrets.token_urlsafe)

## 🛠 Сервисные функции (вызовы на данный момент не реализованы)

### `app/services/session_service.py`

**`cleanup_expired_sessions(db)`**
- **Назначение:** Удаляет все просроченные сессии из базы данных
- **Зачем:** Предотвращает разрастание таблицы `sessions`, освобождает место
- **Когда использовать:** Запускать по расписанию (cron) раз в сутки/неделю
- **Пример:**
  ```python
  @app.on_event("startup")
  async def cleanup():
      async with AsyncSessionLocal() as db:
          deleted = await cleanup_expired_sessions(db)
          logger.info(f"Deleted {deleted} expired sessions")
  ```

**`delete_all_user_sessions(db, user_id)`**
- **Назначение:** Удаляет ВСЕ сессии конкретного пользователя
- **Зачем:** Принудительный логаут на всех устройствах (например, при смене пароля или подозрительной активности)
- **Когда использовать:** В админ-панели или при сбросе доступа
- **Пример:**
  ```python
  @router.post("/admin/users/{user_id}/logout-all")
  async def logout_all(user_id: int, db: AsyncSession = Depends(get_async_db)):
      count = await delete_all_user_sessions(db, user_id)
      return {"deleted_sessions": count}
  ```

## 📞 Авторизация по звонку (Check Call) — ✅ НОВОЕ в v2.0.0

### Описание метода

Альтернативный бесплатный способ верификации пользователя без ввода SMS-кода. Вместо отправки цифрового кода система совершает автоматический звонок на указанный номер телефона. Пользователю достаточно просто дать гудок — верификация происходит автоматически.

### 💰 Преимущества Check Call

- **Время:** Сокращено на 10-30 сек
- **Деньги:** Бесплатно для бизнеса и пользователей
- **Удобство:** Пользователям не нужно вводить код

### Как это работает

1. Пользователь вводит имя и телефон
2. Нажимает кнопку "Продолжить"
3. Получает кнопку с номером телефона для бесплатного звонка в SMS.ru
4. После успешного вызова отправляется webhook о successful вызове
5. Система автоматически верифицирует пользователя
6. Фронтенд опрашивает статус звонка и перенаправляет на карту с QR
7. Сессия сохраняется для повторного входа в приложение

### Технические детали

**Интеграция с SMS.ru:**
- **Endpoint для звонка:** `https://sms.ru/call/initiate`
- **Webhook:** `/api/auth/webhook/sms-ru` (получает статус звонка)
- **Polling:** `/api/auth/check-call-status` (фронтенд опрашивает статус каждые 2 сек)

**Настройка в `.env`:**
```env
SMS_API_KEY=your_api_id  # API ID из личного кабинета SMS.ru
SMS_TEST_MODE=True       # True = тестовый режим (симуляция звонка), False = реальные звонки
```

**Требования:**
- ✅ Аккаунт в SMS.ru, интеграция webhook
- ✅ Активированный API ключ с услугой "Check Call"
- ✅ Номер телефона в формате `+7XXXXXXXXXX`

### Отличия от SMS-верификации

| Параметр | SMS | Check Call |
|----------|-----|------------|
| Скорость | ~10-30 сек (ожидание + ввод) | ~5 сек (только звонок) |
| Удобство | Нужно вводить код | Нужно совершить вызов |
| Стоимость | ~2-3 руб/SMS | бесплатно |
| Надёжность | Возможна задержка SMS | Мгновенная доставка |
| Тестовый режим | Код "0000" | Симуляция ~5 сек |

## 📊 Админ-панель

### Доступ

```
GET /admin          # Список пользователей
GET /admin/export   # Экспорт в CSV
```

### Функционал

- Просмотр всех зарегистрированных пользователей
- Статус верификации (верифицирован / ожидает)
- Дата регистрации
- **Поиск по телефону** (мгновенный поиск по всей базе с задержкой 300мс)
- **Пагинация** (50 пользователей на странице, навигация: ← Предыдущая / Следующая →)
- **Подсчёт верифицированных** (общее количество по всей базе)
- Экспорт базы для email/SMS-рассылок

### Особенности

- Поиск работает по всей базе пользователей, а не только среди отображённых
- При вводе номера телефона таблица обновляется автоматически (AJAX + debounce)
- Пагинация отключается при поиске (показываются все найденные результаты)
- Индикатор загрузки во время поиска
